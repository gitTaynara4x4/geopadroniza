2. Escolha o Modo

Ultra-rápido
Motor local reforçado. Ideal para lotes grandes quando você quer velocidade com correção inteligente sem web.
100% Local Realoca e limpa

Neste modo, o sistema executa:

  • Limpeza e padronização de texto
  • Normalização de logradouros, número, cidade, bairro, UF e CEP
  • Reaproveitamento de valores em coluna errada quando a identificação for segura
  • Tentativa por CEP válido na base local
  • Se o CEP estiver ruim, tentativa por endereço na base local
  • Se ainda falhar, tentativa de CEP aproximado da cidade/UF na base local
  • Limpeza do valor inválido quando não houver recuperação confiável
Restrição: não realiza chamadas web. É o modo mais rápido, mas já faz correção local forte.
Rápido
No backend atual, usa o mesmo motor local do Ultra. É uma opção intermediária de uso, sem consultas web.
Base Local Sem Web

Neste modo, o sistema executa:

  • As mesmas etapas locais do modo Ultra no backend atual
  • Padronização e tentativa de correção sem usar ViaCEP ou geocodificação
  • Busca por CEP, endereço e fallback por cidade/UF usando apenas a base local
Observação: com o backend atual, Ultra e Rápido usam a mesma lógica local. O modo Completo é quem adiciona validação web.
Completo
Usa toda a base local e ainda tenta validação externa. Indicado para fechamento e revisão mais profunda.
Base Local + Web Máxima Revisão

Neste modo, o sistema executa:

  • Tudo que já é feito pelo motor local
  • Tentativa adicional por ViaCEP quando necessário
  • Busca complementar por endereço via web
  • Geocodificação via OSM/Nominatim quando aplicável
  • Revisão mais profunda para lotes menores e finalização
Aviso: é o modo mais lento, porque adiciona consultas externas após a etapa local.

Comportamento do Motor

Motor local reforçado: normaliza, tenta corrigir CEP/endereço, move dados para a coluna certa quando possível e limpa o que não for confiável.

Como funciona

Sem geo_*: não cria colunas extras. O ajuste acontece nas colunas originais.
Realocação inteligente: se entrar UF na cidade, CEP no bairro ou cidade no CEP, o sistema tenta mover para a coluna certa.
Limpeza segura: quando um valor não for confiável e não puder ser recuperado, ele é limpo em vez de ficar como lixo.
💡 Dica de Fluxo: use Ultra-rápido no lote inteiro para uma passada forte local. Use Completo só nas linhas ou arquivos que ainda precisarem de validação externa.
